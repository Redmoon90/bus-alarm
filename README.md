# 🚌 버스 하차 알리미 — 네이티브 앱 (안드로이드)

웹앱의 모든 기능(정류장 검색·앞차/뒤차 도착정보·노선 경로·현재 위치)에
**화면이 꺼져 있어도 작동하는 백그라운드 200m 하차 알람**을 더한 안드로이드 앱입니다.

이 폴더를 GitHub에 올리면 **자동으로 APK가 만들어집니다.** PC에 개발도구를 설치할 필요가 없습니다.

---

## 📦 APK 만들기 (클라우드 빌드) — 단계별

### 1. GitHub 무료 계정 만들기
- https://github.com 접속 → **Sign up** → 이메일로 가입 (무료)

### 2. 새 저장소(repository) 만들기
- 로그인 후 오른쪽 위 **+** → **New repository**
- Repository name: `bus-alarm` (아무 이름 가능)
- **Public** 또는 **Private** 아무거나 선택
- **Create repository** 클릭

### 3. 이 폴더의 파일들을 업로드
- 만든 저장소 화면에서 **Add file ▾ → Upload files** 클릭
- 윈도우 탐색기에서 **`bus-alarm-native` 폴더 안의 모든 항목**을 선택해서
  (`www`, `.github`, `package.json`, `capacitor.config.json`, `.gitignore`, `README.md`)
  드래그해서 올립니다.
  > ⚠️ `.github` 폴더가 꼭 포함되어야 합니다 (자동 빌드 설정이 들어있음).
  > 윈도우에서 점(.)으로 시작하는 폴더도 그대로 보이니 함께 선택하세요.
- 아래 **Commit changes** 버튼 클릭

### 4. 자동 빌드 기다리기 (약 3~5분)
- 위쪽 메뉴 **Actions** 탭 클릭
- **Build Android APK** 작업이 노란 점(진행중) → 초록 체크(완료)로 바뀝니다
- 빨간 X가 뜨면 작업을 눌러 오류를 확인 (보통 다시 실행하면 해결)

### 5. APK 다운로드
- 완료된 작업(초록 체크)을 클릭
- 맨 아래 **Artifacts** → **버스알리미-apk** 클릭 → zip 다운로드
- zip 안에 `app-debug.apk` 가 들어 있습니다

### 6. 갤럭시 폰에 설치
- `app-debug.apk` 를 폰으로 옮기기 (카톡 "나에게 보내기", USB, 구글드라이브 등)
- 폰에서 파일 탭 → 설치
- "출처를 알 수 없는 앱" 경고가 뜨면 **설정 → 허용** 후 설치
  (개인이 직접 만든 앱이라 정상입니다)

## ⚙️ 첫 실행 시 권한 설정 (중요!)

화면이 꺼져도 알림이 오게 하려면 아래 권한이 필요합니다:

1. **위치 권한 → "항상 허용"**
   - 앱이 처음 위치를 요청할 때 "앱 사용 중에만 허용"이 아니라
     **"항상 허용"** 으로 설정해야 화면 꺼짐 상태에서도 추적됩니다.
   - (안드로이드 정책상 처음엔 "사용 중에만"만 보일 수 있어요 →
     설정 → 애플리케이션 → 버스 하차 알리미 → 권한 → 위치 → **항상 허용**)

2. **알림 권한 → "허용"**

3. **배터리 최적화 제외 (권장)**
   - 설정 → 애플리케이션 → 버스 하차 알리미 → 배터리 → **"제한 없음"**
   - 이걸 안 하면 폰이 절전하면서 알림이 늦거나 안 올 수 있습니다.

## 🔄 앱을 수정했을 때
`www/index.html` 등을 고친 뒤 GitHub에 다시 업로드(Commit)하면
**자동으로 새 APK가 빌드**됩니다. (Actions 탭에서 다시 다운로드)

수동으로 빌드하려면: **Actions 탭 → Build Android APK → Run workflow**
